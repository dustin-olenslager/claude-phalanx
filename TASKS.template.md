# TASKS

Backlog for the autonomous loop. Copy this into a repo root as TASKS.md.
- Top unchecked `- [ ]` is picked next. Order = priority.
- One line = one task. Keep each independently shippable (own branch/PR).
- Add acceptance criteria after a `--` so the orchestrator knows when it's done.
- `- [x]` = done (the loop checks these off itself).

## Tasks
- [ ] Example: add input validation to the signup form -- reject empty email, show inline error, unit test covers both

## Notes (free text, ignored by the picker)
- Anything here is context, not a task. No `- [ ]` = not picked up.
